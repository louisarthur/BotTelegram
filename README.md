# BotTelegram
Bot Telegram

1 - Introdução 
Em virtude das constantes mudanças tecnológicas que a sociedade vem enfrentando, diversas aplicações móveis surgem diariamente com intuito de auxiliar no desenvolvimento social e econômico. Esses aplicativos objetivam atender uma necessidade que normalmente não é atendida atualmente pelos mais diversos motivos. 
Dentre essas necessidade pode-se citar a ausência de conexão/comunicação eficiente e eficaz entre pessoas, partindo dessa pressuposição surgiu o aplicativo de mensagens instantâneas Telegram em 2013 desenvolvido pelos irmãos Nikolai e Pavel Durov, apesar de nesse mesmo período o aplicativo de mensagens WhatsApp estar em um crescente ascensão de usuários.
O Telegram é um serviço de mensagens instantâneas baseado na nuvem, atendendo os mais diversos tipos de dispositivos, computadores e também aplicação web. Os clientes do Telegram possuem código aberto e o serviço também fornece APIs para os desenvolvedores independentes, este fato possibilita a criação de diversas formas de modificação e automação da plataforma do Telegram, viabilizando a criação de robôs por exemplo.

2 - Objetivo
Partindo da possibilidade supracitada a disciplina de Linguagem de Programação II teve como seu projeto final o desenvolvimento de um robô (bot)  no Telegram no qual seria necessário fazer a administração de patrimônios. fazendo-se necessário elementos como: cadastrar, atualizar e manter os bens de uma determinada empresa quanto a sua localização, categoria, descrição, código e número de registro (tombamento). 
Para executar a demanda do projeto, será esperado o uso dos conhecimentos que foi explicitados na disciplina que são: Organização de pacotes,  utilização de bibliotecas externas, documentação JavaDoc, herança, polimorfismo, classe abstrata, interfaces, classes para tratamento de exceções, utilizar pelo menos 2 padrões de projetos e testes unitários.
Para fazer esse projeto o bot obrigatoriamente teria que cumprir os requisitos funcionais do sistema que são: Cadastrar localização contendo um nome e uma descrição, cadastrar uma categoria contendo um código, nome e descrição, cadastrar um bem contendo um código, descrição, nome, localização e categoria, o bot também terá que ser capaz de fazer a listagem das categorias, localizações e bens cadastrados no sistema, fazer buscas por nome, por código e por descrição, movimentar o bem para um outro local e além de gerar um relatório listando todos os bens agrupados por localização e depois por categoria e depois por nome (agrupa primeiro por localização, depois por categoria e depois por nome) ordenados alfabeticamente.

3 - Desenvolvimento
	O desenvolvimento do projeto foi regido por 2 padrões de projeto o MVC e o STATE, o primeiro é um acrônimo para Model-View-Controller que de acordo com David (2008) esse tipo de arquitetura fomenta o reuso de código, melhora a performance devido a separação em camadas e melhora a produtividade, de fato com o decorrer do desenvolvimento do projeto, foi perceptível que o uso desse padrão de projeto ajudou em termos de produtividade e reutilização de código. 
O outro padrão de projeto é o State que permite que um objeto altere o seu comportamento quando o seu estado interno muda, esse padrão encapsula os estados em classes separadas e delega as tarefas para o objeto que representa o estado atual, nós sabemos que os comportamentos mudam juntamente com o estado interno.
O projeto foi desenvolvido na linguagem de programação JAVA, utilizando uma Programação Orientada a Objeto, analisando os requisitos funcionais separamos o nosso programa em 3 models, 3 controllers, uma classe main para inicialização da API do Telegram, uma classe de manipulação dos comando que virão a ser recebidos pelo bot e uma máquina de estados.
Fragmentamos o programa em 3 models, estes são: Asset (Bem), Location (localização) e Category (categoria), também fizemos um controller para cada model, ressaltando que a ausência de View que consta no padrão MVC se dá pelo fato que não utilizaremos um interface gráfica ou uma interface para web o usuário utilizará a própria interface do Telegram para ter acesso aos serviços do bot.
Para fazermos a integração com o bot utilizamos o arquivo jar que o telegram disponibiliza em seu site, e utilizamos somente 1 método para recebimento de mensagens do Telegram, fazendo a administração por uma máquina de estados que facilitou o projeto.

4 - Principais dificuldades do desenvolvimento
	O projeto proposto pela disciplina foi relativamente tranquilo de se fazer, a maioria das dificuldades encontradas no projeto foi relacionar o uso das classes abstratas e interface no projeto, visto que o projeto é bem pequeno foi necessário desprender algumas  horas de ócio para pensar sobre a utilização dessas classes. Outra dificuldade encontrada foi na utilização de uma IDE começamos programando no eclipse, entretanto migramos para o IntelliJ, visto que se demonstrou mais fácil implementar por essa IDE.
	Uma dificuldade que foi preciso ter mais atenção foi ter a percepção que a API e o programa trabalham de forma assíncrona, foi necessário de um pouco de tempo para entender esse problema.
5 - Conclusão 
	A utilização da API e a implementação do que foi pedido no projeto foi um sucesso, todos os requisitos funcionais e não-funcionais foram cumpridos com êxito. O projeto foi desafiador por propôr o trabalho em cima de uma API, o que é bastante comum no dia a dia de quem trabalha com programação, tornar esse tipo de manipulação mais comum, nos torna cada vez mais programadores preparados para o mercado, o uso dos conteúdos explicitados em sala na aplicação foi bastante simples, pois foi possível perceber com bastante clareza onde deveria ser abordado cada conteúdo.
	O JavaDoc gerado irá explicitar melhor os trechos de código do sistema, visto que foi bem documentado para compreensão de terceiros e fins avaliativos, tornando dispensável introduzir demasiadamente códigos dentro desse relatório.
